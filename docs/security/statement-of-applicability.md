# Statement of Applicability

## Purpose

This Statement of Applicability (SoA) identifies the information security controls from ISO/IEC 27001:2022 Annex A that are applicable to the Chiral Infrastructure as Code platform, along with justification for inclusion or exclusion.

## Scope

Applies to the ISMS scope defined in the Information Security Policy.

## Control Objectives and Controls

### A.5 Information Security Policies
- **Applicable**: Yes
- **Justification**: Required for ISMS establishment
- **Implementation**: Information Security Policy document

### A.6 Organization of Information Security
- **Applicable**: Yes
- **Justification**: Essential for security governance
- **Implementation**: Roles and responsibilities defined

### A.7 Human Resource Security
- **Applicable**: Yes (where applicable)
- **Justification**: Personnel handling sensitive information
- **Implementation**: Security awareness training

### A.8 Asset Management
- **Applicable**: Yes
- **Justification**: Protection of information assets
- **Implementation**: Asset inventory and classification

### A.9 Access Control
- **Applicable**: Yes
- **Justification**: Critical for data protection
- **Implementation**: RBAC and MFA

### A.10 Cryptography
- **Applicable**: Yes
- **Justification**: Protection of sensitive data
- **Implementation**: Encryption standards

### A.11 Physical and Environmental Security
- **Applicable**: Partial (cloud-based)
- **Justification**: CSP responsibility for physical security
- **Implementation**: N/A - delegated to CSP

### A.12 Operations Security
- **Applicable**: Yes
- **Justification**: Secure operations
- **Implementation**: Monitoring and backup procedures

### A.13 Communications Security
- **Applicable**: Yes
- **Justification**: Secure data transmission
- **Implementation**: TLS encryption

### A.14 System Acquisition, Development and Maintenance
- **Applicable**: Yes
- **Justification**: Secure development lifecycle
- **Implementation**: Security testing and code review

### A.15 Supplier Relationships
- **Applicable**: Yes
- **Justification**: Third-party risk management
- **Implementation**: Vendor assessments

### A.16 Information Security Incident Management
- **Applicable**: Yes
- **Justification**: Incident response capability
- **Implementation**: Incident response plan

### A.17 Information Security Aspects of Business Continuity
- **Applicable**: Yes
- **Justification**: Business continuity
- **Implementation**: Backup and recovery

### A.18 Compliance
- **Applicable**: Yes
- **Justification**: Legal and regulatory compliance
- **Implementation**: Compliance monitoring

## ISO 27017 Additional Controls

All 7 additional controls are applicable for cloud security.

## ISO 27018 Additional Controls

All privacy controls are applicable for PII protection.
